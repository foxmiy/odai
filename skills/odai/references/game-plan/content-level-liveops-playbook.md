# Content Level LiveOps Playbook

## 何时读取

任务主矛盾落在关卡 / 副本 / 任务 / 章节 / 引导、活动 / 版本 / 赛季 / 长期运营、叙事 / 世界观 / 角色内容、内容产能 / 复用 / 更新节奏时读取。

## 执行顺序

1. 定目标：拉什么指标、制造什么体验、解决什么版本问题。
2. 定结构：章节 / 关卡 / 活动 / 赛季如何分层，奖励与门槛如何挂。
3. 定节奏：首次体验、中期重复、后期追赶、版本更新、资源回收。
4. 定产能：哪些复用，哪些新做，长期成本能否承受。

## 检查表

关卡 / 任务：
- 玩家目标、失败条件、反馈节奏。
- 关卡结构、事件编排、难度曲线、奖励投放。
- 新手引导、教学节点、卡点保护。
- 可复用模板与一次性内容比例。

活动 / 版本 / 赛季：
- 指标目标或待修问题。
- 开放节奏、资源回收、奖励结构、疲劳控制。
- 与原系统、商业结构、数值节奏的耦合。
- 老玩家、新玩家、回流玩家影响。
- 结束后可复用资产与新增技术 / 内容债。

叙事 / 世界观：
- 题材定位与内容功能：沉浸、转化、角色认同或版本承接。
- 世界规则、角色关系、剧情推进、分支控制。
- 文本、演出、立绘、任务或事件系统的协同。
- 内容产能、复用机制、跨版本延展。

## 最小交付

按主矛盾输出：关卡 / 副本方案（目标、结构、事件、节奏、奖励、风险）；活动 / 版本 / 赛季方案（指标、内容节奏、资源回收、复用、风险）；叙事 / 世界观草案（题材、角色关系、剧情功能、产能、承接）；新手引导方案（教学节点、卡点保护、反馈节奏、早期留存目标）。

不要只交地图概念、活动包装或设定词条；必须写玩家实际经历的节奏，以及它如何服务玩法、版本和商业目标。
