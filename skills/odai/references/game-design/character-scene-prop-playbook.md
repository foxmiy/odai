# Character Scene Prop Playbook

## 何时读取

由 `game-design` 触发；覆盖角色 / 场景 / 道具的视觉方向、识别性、一致性与资产产能。

## 执行顺序

1. 定题材、目标玩家、观看距离：什么设备、视距、频次下看这些资产。
2. 定识别锚点：阵营、职业、稀有度、危险性或功能性靠什么一眼识别。
3. 定层级：主角、敌人、NPC、场景、道具谁抢焦点，谁做支撑。
4. 定验收：识别性、一致性、复用性、产能边界是否成立。

## 检查表

角色视觉：题材定位、阵营差异、形体语言；色彩系统、材质倾向、细节密度；主角 / 敌人 / NPC 在剪影和远视距下的区分；稀有度、职业或势力的视觉层级。

场景与道具：世界规则、区域主题、功能区分；建筑、机关、装置、可互动物的功能可读性；前中后景分工与探索引导；道具、载具、资源点与战斗判定信息是否互相干扰。

资产与产能：哪些必须高精度，哪些可模板化 / 模块化；角色、场景、道具之间可复用的规则和部件；长期运营或多版本更新时最容易失控的视觉债。

## 最小交付

按主矛盾输出：角色视觉方向（题材、形体、色彩、识别性、一致性、风险）；场景 / 道具视觉方向（世界规则、层级、功能可读性、复用、风险）；统一规则（阵营、材质、视距、细节密度、资产约束）。
